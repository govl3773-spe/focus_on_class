# 웹캠 기반 탐지 프로토타입 재구성 계획

## 1. 현재 목표
- CNN 학습 이전 단계의 프로토타입 구성
- 웹캠 기반 실시간 탐지 파이프라인 점검
- 분류 모델 연결 전, 탐지/전처리 요소 우선 검증
- Jupyter Notebook 환경에 맞는 셀 분할형 코드 구성

## 2. 이번 단계에서 구현할 범위
- 웹캠 입력 연결
- 좌우 반전 적용
- MediaPipe 기반 얼굴 탐지
- 블루 오버레이 세그멘테이션 시각화
- 손(Hand) 탐지 모델 추가 연결
- 얼굴/손 탐지 결과 화면 출력
- 추후 데이터 수집 및 CNN 입력용 전처리 확장 가능 구조 확보

## 3. 이번 단계에서 아직 하지 않을 것
- CNN 학습 코드 구현
- 3개 클래스(attentive / drowsy / inattentive) 분류
- 라벨 저장 로직 고도화
- 실시간 상태 판정 로직 확정
- KNN 등 전통 ML 분류기 연결

## 4. 설계 방향
### 4.1 얼굴 탐지기 선택 방향
- 프로젝트 목적 적합성 우선
- 화상수업/웹캠 환경에서 안정적으로 얼굴 위치를 찾을 수 있는 방식 채택
- 얼굴 위치 검출 + 후속 crop 전처리 연계 가능 구조
- 필요 시 Face Detection, Face Landmarker, Image Segmenter 조합 검토

### 4.2 좌우 반전 적용
- 웹캠 미리보기의 사용자 경험 개선
- 거울 화면 형태 제공
- 탐지 및 시각화 결과도 반전된 화면 기준으로 출력

### 4.3 블루 오버레이 세그멘테이션 추가
- 사람 영역 시각적 강조
- 프로토타입 완성도 향상
- 추후 상반신/얼굴 중심 관심영역 설정 참고 가능

### 4.4 Hand 모델 추가
- 보유 중인 hand 모델 연결 가능성 점검
- 손 영역 탐지 및 랜드마크 표시
- 집중/비집중 상태 확장 판단 요소 후보로 활용 가능성 확인
- 현재 단계에서는 “탐지가 잘 되는지” 수준까지만 검증

## 5. 구현 흐름
- 웹캠 프레임 입력
- 좌우 반전
- BGR → RGB 변환
- MediaPipe 입력 변환
- 세그멘테이션 수행
- 얼굴 탐지 수행
- 손 탐지 수행
- 세그멘테이션 블루 오버레이 적용
- 얼굴 박스/손 랜드마크 시각화
- 최종 화면 출력
